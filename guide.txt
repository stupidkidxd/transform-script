Найти ID объекта по IMEI: Сначала вам нужно найти ID объекта, используя его IMEI. Вы можете использовать метод core/search_items для поиска объектов по их уникальному ID (которым часто является IMEI).

Пример запроса для поиска объекта по IMEI:

    https://hst-api.wialon.com/wialon/ajax.html?svc=core/search_items&params={"spec":{"itemsType":"avl_unit","propName":"sys_unique_id","propValueMask":"ВАШ_IMEI_УСТРОЙСТВА","sortType":"sys_name"},"force":1,"flags":1,"from":0,"to":0}&sid=ВАШ_SID
    ```

    Замените `ВАШ_IMEI_УСТРОЙСТВА` на фактический IMEI и `ВАШ_SID` на ваш идентификатор сессии. В ответе вы получите список объектов, соответствующих IMEI, и сможете извлечь их ID.

2.  **Экспортировать свойства объекта в WLP:**
    После того как вы получили ID объекта, вы можете использовать метод `exchange/export_json` для экспорта его свойств в WLP-файл. Вам нужно будет сформировать JSON-структуру, которая указывает, какие свойства объекта вы хотите экспортировать.

    Пример запроса для экспорта свойств объекта с определенным ID в WLP-файл:
    
https://hst-api.wialon.com/wialon/ajax.html?svc=exchange/export_json&params={"fileName":"unit_properties.wlp","json":{"units":[{"id":ID_ВАШЕГО_ОБЪЕКТА,"props":["general","sensors","commands","custom_fields","fuel_consumption","maintenance","eco_driving","profile","icon","advanced"]}]}}&sid=ВАШ_SID
```

Замените `ID_ВАШЕГО_ОБЪЕКТА` на полученный ID объекта и `ВАШ_SID` на ваш идентификатор сессии. В массиве `props` вы можете указать, какие именно свойства объекта вы хотите включить в WLP-файл.
Важные замечания:

Для выполнения этих запросов у вашего пользователя должны быть соответствующие права доступа к объектам и функции импорта/экспорта.
Метод exchange/export_json создает WLP-файл, который затем можно загрузить.
